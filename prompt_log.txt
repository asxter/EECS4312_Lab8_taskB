The system manages registration for a single event with limited capacity. Functional Requirements:

FR2: The system shall automatically handle registration conflicts without requiring manual user intervention.
FR3: The system shall automatically promote the earliest waitlisted user when a registered user cancels.
FR4: The system shall preserve first-in, first-out order in the waitlist under all valid operations.
FR5: The system shall return exactly one operation result and one updated registration state for each completed operation.
FR6: The system shall return a clear result after each operation.
FR7: The system shall return the updated registered list and updated waitlist after each operation.
FR8: The system shall allow a user to query their current status.
FR9: The system shall explicitly report when a user is not found in the system.
FR10: The system shall explicitly report when a registration request is rejected as a duplicate.
FR11: The system shall explicitly report the result of a cancellation or waitlist promotion.
FR12: The system shall prevent duplicate registrations.
FR13: The system shall preserve a consistent system state after every operation and shall not produce contradictory outputs for the same state.
FR14: The system shall process one operation at a time.
FR15: The system shall support multiple cancellations in sequence and other edge cases without silent failure.
FR16: The system shall allow users to register again after canceling.
FR17: The system shall support a capacity value equal to zero, and when capacity is zero, all registering users shall be placed in the waitlist.
FR18: If the same initial state and the same sequence of operations are applied more than once, the system shall produce the same registered list, the same waitlist, and the same operation results each time.
